Installation

Installing MS-AutoQC is relatively easy and straightforward. You’ll need to:

  1. Install Python
  2. Install MS-AutoQC
  3. Download MS-DIAL
  4. Install .NET Framework
  5. Install MSConvert

If you run into trouble or have any questions, please feel free to open a new issue on GitHub. You can also visit the Frequently Asked Questions to see if your issue has been documented already.

Important note

While, to our knowledge, installation of the following software packages on data acquisition computers comes at no risk, it is possible for a wary user to install MS-AutoQC on a separate computer.

Because MS-AutoQC only needs access to the data acquisition path (i.e. the folder where your MS data is being collected), you can install all of the packages on a different computer. The only requirement is that the instrument computer must be mounted as a network drive on the MS-AutoQC installation computer.

If you don’t have any experience with mapping a network drive in Windows, consult your IT department or check out this tutorial on Microsoft Support.

We have not tested this functionality ourselves. If you plan to test this out, please let us know how it goes!

1. Install Python

If you don’t have Python installed already, you can download and install the latest release of Python 3 for Windows or Mac (which is v3.11 at the time of this writing).

2. Install MS-AutoQC

There are several ways to install MS-AutoQC. Please choose what works best for you.

Option 1: Install with pip (recommended)

To install MS-AutoQC using pip, simply enter the following command in your terminal:

py -m pip install ms-autoqc

MS-AutoQC and all required dependencies will be installed automatically. This may take a minute or two. (If you’re curious, you can find a list of these dependencies here.)

Option 2: Install manually

If you prefer not to use Python’s package manager – or if your LC-MS instrument computer does not have an internet connection – you can manually download and install MS-AutoQC by following the steps below.

  1. Download MS-AutoQC v1.0.0 from GitHub.
  2. Extract the ZIP file to a folder of your choice.
  3. Open a terminal window and enter:
cd <path-to-folder>
py -m pip install .

That’s it! MS-AutoQC and its dependencies will be installed automatically.

Option 3: Install in a virtual environment

If you would like to protect the integrity of other Python packages or versions installed on your instrument computer, you can opt to install MS-AutoQC in a virtual environment.

Choose a folder that you would like to store the virtual environment in – denoted by <path-to-venv-folder below – and then enter the following commands in a terminal window:

py -m venv <path-to-venv-folder>
<path-to-venv-folder>\Scripts\activate
py -m pip install ms-autoqc

That’s it! MS-AutoQC and its dependencies will be installed in your virtual environment.

Note: Every time you launch MS-AutoQC, you will need to activate your virtual environment first. To do this, just type:

<path-to-venv-folder>\Scripts\activate

Start MS-AutoQC

To launch MS-AutoQC, simply open a terminal window and type:

ms_autoqc

If you see the screen below, then MS-AutoQC was installed properly!

If you run into any problems with installing or opening the app, please visit the FAQ on installing MS-AutoQC.

Important note

At this point, if you are installing MS-AutoQC only to sign in to your workspace and view the QC dashboard, you can move on to the Quickstart guide to learn how to get started using MS-AutoQC.

If you are installing MS-AutoQC on an instrument computer for the first time, continue with installation of MS-DIAL and MSConvert below.

3. Download MS-DIAL

You can download the latest version of MS-DIAL 4 (which is v4.9.2 at the time of this writing). Once the download has completed, extract the ZIP file into a directory of your choice.

Remember this directory path! You will need to provide it to MS-AutoQC later.

4. Install .NET Framework

Versions 4.7.2 (or newer) of the .NET Framework is required to install MSConvert. You download and install the .NET Framework from the official Microsoft website.

Once the installation is complete, don’t forget to restart your computer if needed.

5. Install MSConvert

You can download and install MSConvert from the ProteoWizard website. For Platform, select Windows 32-bit installer (able to convert vendor files).

6. Celebrate!

That’s it! If you’ve reached this point, you’re one giant step closer to using MS-AutoQC for your LC-MS instrument runs.

When you’re ready, visit the quickstart guide to learn how to configure your MS-AutoQC workspace and set up your first QC job.
